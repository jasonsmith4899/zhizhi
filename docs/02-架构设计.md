# 智知 AI知识库 - 架构设计文档

## 1. 系统架构

### 1.1 整体架构

```
┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│  Vue3 管理台  │  │ 微信小程序   │  │  API 调用    │
│  :3000      │  │             │  │  (API Key)  │
└──────┬──────┘  └──────┬──────┘  └──────┬──────┘
       │                │                │
       └────────────────┼────────────────┘
                        │
               ┌────────▼────────┐
               │  Nginx / API GW │
               └────────┬────────┘
                        │
               ┌────────▼────────┐
               │  Spring Boot    │
               │  后端 :8080     │
               │  ┌────────────┐ │
               │  │JWT Filter  │ │
               │  │Tenant拦截器│ │
               │  │Rate Limit  │ │
               │  └────────────┘ │
               └───┬────────┬───┘
                   │        │
        ┌──────────▼──┐  ┌──▼──────────┐
        │ PostgreSQL  │  │ DeepSeek AI │
        │ + pgvector  │  │ + SiliconFlow│
        │ :5432       │  │ (Embedding) │
        └─────────────┘  └─────────────┘
```

### 1.2 分层架构

- **Controller层**: REST API接口，参数校验，调用Service
- **Service层**: 业务逻辑，事务管理
- **Repository层**: 数据访问，JPA + 原生SQL
- **Common层**: JWT工具、租户上下文、全局异常、限流

### 1.3 多租户隔离机制

1. **认证**: JWT Token 或 API Key → 提取userId
2. **租户解析**: TenantInterceptor从tenant_members表查询用户的tenantId
3. **上下文传播**: TenantContext (ThreadLocal) 存储当前tenantId
4. **数据隔离**: 所有查询必须包含tenant_id条件
5. **向量隔离**: pgvector通过metadata JSON中的tenant_id字段过滤

## 2. RAG架构

### 2.1 文档处理流程

```
上传文档 → Tika解析 → 文本切片(500字/100字重叠) → SiliconFlow BGE-M3向量化 → 存入pgvector
```

### 2.2 问答流程

```
用户提问 → buildRagContext() → pgvector相似度搜索(TopK=5, 阈值0.7) 
→ 构建增强Prompt → DeepSeek LLM生成回答 → 返回回答+来源引用
```

### 2.3 对话记忆

- 使用Spring AI的ChatMemory
- 按sessionId关联对话上下文
- 保留最近10条消息作为上下文

### 2.4 流式对话

- SSE (Server-Sent Events) 实现流式响应
- 前端使用fetch + ReadableStream消费
- AbortController支持取消，60秒超时保护

## 3. 安全架构

### 3.1 认证方式

| 方式 | 用途 | 有效期 |
|------|------|--------|
| JWT Access Token | Web管理台认证 | 2小时 |
| JWT Refresh Token | Token刷新 | 7天 |
| API Key | 第三方API调用 | 永久(可吊销) |

### 3.2 安全措施

- BCrypt密码加密
- JWT签名验证
- CORS跨域配置
- 接口限流(10次/分钟)
- 文件大小限制(20MB)
- 输入校验(Bean Validation)
- 全局异常处理(不泄露内部信息)

### 3.3 DTO隔离

Controller返回DTO而非Entity，隐藏内部字段(tenantId, userId等)：
- UserDTO, ConversationDTO, KnowledgeBaseDTO
- TenantDTO, TenantMemberDTO, MessageDTO, DocumentListDTO

## 4. 前端架构

### 4.1 状态管理

- **auth store**: 用户认证状态、token管理
- **chat store**: 对话消息、sessionId、选中知识库
  - localStorage持久化(限制50条)
  - 跨标签页storage事件同步

### 4.2 Chat组件拆分

- ChatHeader: 知识库选择器、RAG标签
- MessageList: 消息列表、自动滚动
- MessageBubble: 单条消息、推理链路
- ChatInput: 输入框、发送按钮
- SourcePreview: 文档预览弹窗
- useChatStream: SSE流式composable

### 4.3 路由守卫

- 检查token存在性
- 解析JWT exp字段验证过期
- 过期自动logout并跳转登录
- 已登录用户访问/login重定向首页

## 5. 可扩展性分析

### 5.1 换AI模型
- 修改application.yml中的base-url和model即可
- 当前支持任何OpenAI兼容API

### 5.2 换向量存储
- Spring AI VectorStore抽象层
- 需修改application.yml中的vectorstore配置

### 5.3 换文件格式
- Tika自动检测格式
- 支持PDF/TXT/MD，可扩展Word/Excel等
