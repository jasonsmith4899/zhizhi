## 5. 部署流程

### 5.1 代码同步
```bash
# 本地打包
cd /home/mason/zhizhi
tar czf /tmp/zhizhi-deploy.tar.gz --exclude='.git' --exclude='node_modules' --exclude='target' --exclude='data' .

# 上传到服务器
scp -i ~/.ssh/server_key.pem /tmp/zhizhi-deploy.tar.gz root@119.29.112.227:/tmp/

# 服务器解压
ssh -i ~/.ssh/server_key.pem root@119.29.112.227 "cd /opt/zhizhi && tar xzf /tmp/zhizhi-deploy.tar.gz"
```

### 5.2 构建部署
```bash
# 重建并启动(推荐，利用Docker缓存)
ssh -i ~/.ssh/server_key.pem root@119.29.112.227 \
  "cd /opt/zhizhi && docker compose -f docker-compose.server.yml up -d --build backend frontend"

# 强制全量重建(慢，仅在依赖变更时使用)
ssh -i ~/.ssh/server_key.pem root@119.29.112.227 \
  "cd /opt/zhizhi && docker compose -f docker-compose.server.yml build --no-cache backend"
```

### 5.3 数据库迁移
```bash
# Flyway自动迁移(Spring Boot启动时执行)
# 如需手动修复:
docker exec zhizhi-postgres psql -U postgres -d zhizhi -c "SQL语句"
```

### 5.4 健康检查
```bash
curl http://119.29.112.227:8080/api/v1/health
curl -o /dev/null -w "%{http_code}" http://119.29.112.227:3000/
```

## 6. 配置文件

### 6.1 config/application.yml (服务器覆盖)
位于 /opt/zhizhi/config/application.yml，通过Docker volume挂载到容器 /app/config/

关键配置:
- spring.ai.openai: DeepSeek聊天模型配置
- spring.ai.openai.embedding: SiliconFlow BGE-M3嵌入模型
- spring.ai.vectorstore.pgvector: 向量存储(维度1024, HNSW索引)
- spring.flyway: 数据库迁移(validate-on-migrate: false)

### 6.2 docker-compose.server.yml
后端通过 volume `./config:/app/config:ro` 挂载配置

## 7. 常见问题

### 7.1 Flyway校验和不匹配
原因: 迁移文件被修改
解决: 在数据库中更新flyway_schema_history的checksum，或设置validate-on-migrate: false

### 7.2 Schema验证失败(missing column)
原因: Entity新增字段但数据库未同步
解决: ALTER TABLE ADD COLUMN IF NOT EXISTS

### 7.3 向量维度不匹配
原因: embedding模型输出维度与数据库向量列维度不一致
解决: 确保application.yml中dimensions=1024与BGE-M3模型一致

### 7.4 Embedding URL双重/v1
原因: Spring AI的baseUrl会自动追加/v1
解决: 配置时不要包含/v1后缀
